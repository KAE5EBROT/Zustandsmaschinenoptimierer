smtable::fstate smtable::link ( elementlist  inputs,

elementlist  outputs,

Insert transition into table.

Link states on specific input patterns and set outputs. inputs is a vector of strings, containing the exact names of inputs which values are specified in inputval. The size of inputs must match the length of inputval. The values can be '0', '1' or 'x'. If a pair of input and its value is not mentioned, its treated as 'x'.

This function is the heart of the class smtable. It is used to enter a single transition information into the table. It handles ambiguous description of values. They can either be don't cares ('x') or be unmentioned in transition definition.

Parameters
[in]inputsList of input names to be read. Type: vector<string>
[in]inputvalString with values the corresponding inputs should have for transition to trigger. Values can either be '0', '1' or 'x' (don't care). Has to match size of inputs.
[in]srcstateString with name of initial state.
[in]outputsList of output names to be set. Type: vector<string>
[in]outputvalString with values the corresponding outputs should have after transition. Values can either be '0', '1' or 'x' (don't care). Has to match size of outputs.
[in]dststateList of input names. Type: vector<string>
[out]none
Returns
execution status
